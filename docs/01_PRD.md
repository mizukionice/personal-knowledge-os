# 01_PRD — Product Requirements Document

## 1. Vision
**Product Name:** Personal Knowledge OS
**Mission:** ユーザーが所有する専門書・PDF・写真を解析し、AIが理解可能な知識へ変換し、長期的に育成される個人専用Knowledge Baseを構築する。

本サービスはOCRツールではない。目的は「AIに知識を教える」ことである。
価値の中核は文字起こしではなく、複数の情報源を横断して知識を構造化・接続すること（例:「この本のリスク管理はPMBOKの内容と共通」「この概念は先月読んだ本と矛盾」）。

## 2. Problem
汎用AIはインターネット知識には強いが、ユーザー固有の専門書・ノート・資料は知らない。既存RAGツールは「PDF→チャンク→ベクトル検索」で終わり、人間のような知識のネットワーク化（概念の接続・階層・矛盾検出）を行わない。

## 3. Target User
- 当面は開発者本人（個人用ツール）。事業化は動くものができてから検討
- 将来像: エンジニア・研究者・コンサルタント・資格学習者

## 4. Goals（MVP）
写真・PDFをアップロードすると、数分後に:
1. 構造化Markdown（章・図・表の説明を含む）が生成される
2. 概念（Concept）が抽出され、既存知識とリンクされる
3. ハイブリッド検索で横断検索できる
4. 引用付きチャットで自分の蔵書に基づいた回答が得られる

## 5. MVP Scope
### 実装する（マイルストーン順）
- M1: Auth（Supabase）、アップロード（写真/PDF）、ドキュメント一覧
- M2: 処理パイプライン（VLM→Markdown）、ジョブ管理、Markdownビューア
- M3: 知識化（Semantic Chunk / Concept抽出 / Embedding / ハイブリッド検索）
- M4: 引用付きチャット（自分の蔵書を根拠に回答、出典ページまで辿れる）
### 実装しない（将来）
- Knowledge Graph可視化UI、概念マップ、復習・学習支援機能
- 共有・公開・マルチテナント向け課金
- モバイルアプリ

## 6. Success Metrics
- 処理成功率: アップロードした書籍ページの95%以上がエラーなくMarkdown化される
- Markdown品質: 図・表がキャプション+説明文として保持される
- 概念抽出: 専門書1章あたり10個以上の概念が抽出され、既存概念と自動リンクされる
- 検索: 「あの本のあの話」が書名を思い出せなくても見つかる
- チャット: 回答に必ず出典（書名+ページ）が付く
- 処理時間: 100ページの書籍が30分以内に処理完了

## 7. User Story
専門書を読む → スマホで写真を撮る → アップロードする → 数分後、要約・概念・関連知識・Markdownが生成される → 後日チャットすると、その知識を根拠に出典付きで回答する。

## 8. 制約（著作権）
- ユーザー自身が所有する書籍・PDFのみを処理する
- 処理結果はユーザー本人だけが利用する
- コンテンツの再配布・共有・公開機能は実装しない
- 原本画像は処理後もユーザー専用領域に保管し、第三者がアクセスできない設計とする
